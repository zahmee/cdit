# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

CDIT (إبداع التطوير والبرمجة) — a marketing + lead-generation site for a Saudi IT services company selling a ZATCA-compliant accounting system (lite 300 SAR/yr, full 900 SAR/yr). Static multi-page site, `lang="ar" dir="rtl"`, vanilla HTML/CSS/JS, no build system, no package.json, no tests. Deployed on GitHub Pages (`CNAME` → cdit.co, `.nojekyll` present so `_product-screenshots/` underscore folders are served).

There are no build, lint, or test commands. For local preview use `python -m http.server 8765` (port 8765 is in the Cloudflare Workers origin allowlist; VS Code Live Server `:5500` is NOT). A launch config exists in `.claude/launch.json` (`cdit-static`).

## Page inventory (chrome is duplicated in every file)

Root Arabic pages: `index.html`, `services.html`, `products.html`, `portfolio.html`, `about.html`, `contact.html`, `news.html`, `tajribah.html` (ads landing / trial form), plus 4 sector landing pages `sector-contracting.html`, `sector-law.html`, `sector-engineering.html`, `sector-pharmacy.html`. Blog: `blog/index.html` + 3 SEO articles (`zatca-phase2-requirements`, `zatca-fines`, `choose-accounting-software`). English mirror in `en/` (7 pages): `en/index.html` carries the full 2026 redesign (hero-v2, trust strip, pricing, demo, quiz — with LTR override styles in its inline `<style>` block since style.css assumes RTL) and `en/tajribah.html` is the English trial page; the other 5 en pages still have their original layout but carry the AI agent + Free Trial footer link. English pages set `window.CDIT_AI_LANG = 'en'` before `ai-agent.js`; `zatca-demo.js` switches language via `<html lang>`. No English sector pages or blog.

There is **no templating** — header, mobile menu, footer, WhatsApp float are copy-pasted per page. Exception: the 4 sector pages are generated by `scripts/generate-sector-pages.py` (template + per-sector dict) — **edit the script and re-run it, never the sector HTML directly**. Contact info `966502010911` / `info@cdit.co` appears ~15× per page; treat changes as repo-wide find-and-replace.

## Homepage (redesigned 2026-06, conversion-first)

Section order: animated hero (`hero hero-v2`, product-first copy, price line "من 300 ريال", rotating real screenshots in a browser frame, AI-agent invitation card) → trust strip (official ZATCA/Fatoora/VAT/Vision2030 logos + CR/VAT numbers) → stats → pricing (`#pricing`, two `price-card`s, lite is `.featured`) → live ZATCA demo (`#zatca-demo`) → compliance quiz (`#zatca-quiz`) → services → products → why-us → news → CTA. All primary CTAs point to `tajribah.html` (the Google Ads conversion page).

**Live ZATCA demo + quiz = `js/zatca-demo.js`.** The demo builds a real ZATCA phase-1 TLV (tags 1-5) base64 payload and renders it as QR via `js/vendor-qrcode.js` (vendored qrcode-generator 1.5.0, MIT — global `qrcode`, loaded `defer` *before* zatca-demo.js). Everything runs client-side; the copy explicitly promises no data leaves the browser — keep it that way. The quiz is 3 questions → honest result copy (never invent fine amounts or wave dates; max-fine wording "قد تصل إلى 50,000 ريال" is the only number allowed, also in the blog).

## Behavior (`js/main.js`)

- **Header state:** `.transparent` ↔ `.scrolled` on scroll; only `body.page-home` starts transparent.
- **Contact form (`#contact-form`):** posts to the Telegram proxy (type `contact`) — it does NOT open WhatsApp anymore. **Trial form (`#trial-form`, on tajribah + sector pages):** posts type `lead` with gclid/UTM attribution (persisted 30 days in `localStorage`) and fires the Google Ads conversion `window.cditTrackConversion()` (`AW-352580261`). Telegram messages escape `` _ * ` [ `` via `escapeMd` — Markdown parse errors silently drop leads.
- **Scroll reveal:** auto-tags `.service-card, .why-card, .edition-teaser, .post-card, .stat-item, .sector-pain` with `.reveal` + staggered delays; IntersectionObserver adds `.is-visible` once.
- Stats counter, WhatsApp float, lightbox (`.edition-shot` buttons + `#lightbox`), smooth scroll, prayer-times widget (aladhan.com, cached daily).

## AI chat agent (site-wide, `js/ai-agent.js`)

Mounted on **every Arabic page** (root div `#ai-agent-root` + script include) and `en/index.html`. Per-page contextual greeting + a one-per-session teaser bubble (`PAGE_CONTEXT` map; `sector-*.html` reuses the tajribah context; homepage has no teaser — it has the hero card instead). History in `localStorage['cdit_ai_chat_history']` (24h TTL, 20 msgs). Streams SSE from the Cloudflare Worker; synthetic events `{event:'lead_sent'}` (fires the Ads conversion too) and `{error:'stream_failed'}`.

**Backend = Cloudflare Worker `cdit-ai-agent`** (`worker/ai-agent.js`): holds the DeepSeek key, byte-stable system prompt (don't interpolate anything into it — DeepSeek prompt cache), tool `send_lead_to_telegram` calls Telegram API **directly** (worker→worker fetch on `*.workers.dev` fails with error 1042). Rate limit 20 msg/IP/hour via KV. Hard prompt rules that must not regress: never invent prices; collect name+phone+details before firing the lead tool; Saudi dialect persona "مساعد إبداع". Deployment: `worker/AI_AGENT_README.md`.

The second worker `cdit-telegram-proxy` (`worker/telegram-proxy.js`) handles `visit` / `contact` / `lead` notifications (adds country via `request.cf`, silent notification for visits).

## News pipeline

`.github/workflows/news.yml` runs `scripts/generate-news.mjs` every 12h (DeepSeek rewrites HN + TechCrunch headlines into Arabic) → commits `data/news.json` → rendered by `js/news.js` on `news.html` (12 cards) and the homepage (3 cards).

## Honesty rules (owner-approved, apply to all copy)

No fabricated testimonials/clients/stats; prices are public (300/900) and must stay consistent across pages; blog/quiz regulatory claims stay generic and defer to ZATCA as the only official source; demo screenshots are real product shots from `_product-screenshots/`.
